# デモ GIF 絵コンテ（30〜45秒）

記事冒頭・X 告知・README の3箇所で使い回す前提の1本。
メッセージは「テキストを書く → 図が生える → 計画を検査できる → 次の一手が返ってくる」の4拍。
文字が読める尺を確保するため、1シーンを削るくらいなら全体を45秒に伸ばす方を優先する。

## 画面構成

VS Code の左ペインにエディタ、右ペインに PFDSL 拡張のライブプレビュー、下部にターミナル。
フォントは16pt以上、ウィンドウ幅は1280px程度（Zenn・X のインライン再生で文字が潰れない下限）。

## シーン

| # | 尺 | 画面 | 見せたいこと |
|---|-----|------|--------------|
| 1 | 0-8s | エディタに `launch.pfdsl` のエッジ定義を3〜4行、高速タイプで入力。右ペインの図がリアルタイムに生えていく | 「書いた瞬間に図になる」。タイプは2〜3倍速編集でよいが、図の更新タイミングは等速で見せる |
| 2 | 8-15s | 残りの行をペーストで一括追加。図が8ノードの工程図に育つ。1秒静止 | 8行でこの図、という物量感 |
| 3 | 15-25s | わざと `-> requirements` を消して保存。ターミナルで `pfdsl check` → 赤字で `error [V003]: Process 'define_requirements' has no outputs`。直すと check が通る | 「計画そのものを lint できる」。エラー行は1.5秒以上静止 |
| 4 | 25-38s | frontmatter の status を見せ、図が色分きに変わる。`pfdsl ready launch.pfdsl --best` → 推薦が出る。`pfdsl status-set launch.pfdsl design_doc done` → `newly ready: implement`。図の色が変わる | 「次の一手が計算で返ってくる」。`newly ready:` の行は2秒静止 |
| 5 | 38-43s | 黒背景のエンドカード: `Plan as Code — pfdsl` / `github.com/takasek/pfdsl` / `npm i -g @pfdsl/cli` | 着地 |

## 制作メモ

- 収録は [vhs](https://github.com/charmbracelet/vhs)（ターミナル部分をスクリプト収録できる）か、通常の画面録画 + 編集のどちらでも。
  ライブプレビューの「図が生える」瞬間が主役なので、VS Code ごと録るなら画面録画のほうが早い。
- GIF は X だと自動ループ再生される。ループ前提で、シーン5→1の継ぎ目に0.5秒の黒を入れる。
- Zenn 用には同内容の mp4 も書き出しておくとファイルサイズで有利。
- シーン3のエラーメッセージとシーン4の `newly ready:` が2大見せ場。ここだけは絶対に読める尺・サイズにする。
