# ADR-0011: 能力成果物の世代還流

- Status: Accepted
- Date: 2026-06-12

## Context

スキル・ツールチェーンのような「能力」をなす成果物は、自分自身の生産チェーンの
上流プロセス（実践・レビュー・対話）の道具としても使われる。

実例: 生態系 PFD で `toolchain` を実践レビューの入力に通常エッジで追加しようとした際、
当時の位相（レビュー知見が知見外化プロセスへ通常エッジで流れていた）では
レビュー → 知見 → ADR → 運用スキル → 開発 → toolchain → レビュー という
Primary graph の循環が生じた。`check` は循環を検出しないため（issue #4 で lint 化予定）、
規則の明文化が必要になった。

その後のリファクタリングで知見はフィードバックエッジに変わり、同じ通常エッジを
追加しても循環しない位相になった。ここで基準が2択になる:

- R1（構造基準）: `>>?` は循環回避が必要なときだけ使う。機械的で lint と一致するが、
  離れた場所のリファクタリングでエッジ種が変わる不安定さを持つ
- R2（意味基準）: フロー内で生産される能力成果物が自分の生産チェーン上流の
  プロセスに入るときは、位相に関係なく常に `>>?`

## Decision

R2 を採用する。**フロー内で生産される能力成果物（スキル・ツールチェーン・運用スキル等）が、
自分の生産チェーン上流のプロセス（フィードバックエッジ経由の到達を含む）に入力されるときは、
前世代スナップショットとして常に `>>?` で表す。**

適用（生態系 PFD）:

- `skill >>? write_examples` / `skill >>? review_examples` — ラウンド N の実践・レビューは
  ラウンド N-1 の知見から作られたスキルを使う
- `toolchain >>? review_examples`
- `ops_skill >>? develop`

## Rationale

1. **世代の実態に忠実**: プロセスが使うのは「現行世代」の能力であり、その実行が生む
   「次世代」とは別物。実践ラウンド2はラウンド1の知見で改訂されたスキルで書かれた。
   `>>?` はこの世代差を型として表現する（ADR-0002 形態3 = 定常サイクルの適用）
2. **リファクタ耐性**: エッジ種が成果物の意味で決まり、離れた位相の変更で揺れない。
   R1 では知見外化の統合のような無関係なリファクタリングがエッジ種の変更を強制した
3. **読み替え**: `>>?` を「補助入力」でなく「世代をまたぐ入力」と読む。
   「実践はガイドに従う」という主従は弱まらない — 従う相手が前世代だと明示されるだけ

## Consequences

- 能力成果物を上流プロセスに接続するとき、エッジ種の判断は「これは自分の生産
  チェーンに属するか」だけで決まり、現在の位相の循環有無を調べる必要がない
- 循環検出 lint（issue #4）は安全網であり、エッジ種の決定基準ではなくなる
- 規範文書（仕様書等）は能力成果物ではなく本規則の対象外 — 通常入力のまま
