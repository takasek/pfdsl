# ADR-0026: retro トリガーを done イベント駆動に一本化し実行記録を廃止

- Status: Accepted
- Date: 2026-07-09
- Supersedes: ADR-0024

## Context

ADR-0024 は retro 起動条件を計測可能にするため、workflow companion への実行記録追記と、差分計測形の4条件（新規設計決定記録2本以上 / 同一 PFD への修正コミット3回以上 / 設計対話の長期化 / セッションの締め際）を導入した。運用してみると2つの問題が判明した。

1. **並列 worktree との構造的衝突**: 当リポは複数 worktree を並行稼働させる運用が常態（本 ADR 執筆時点で10以上）。実行記録は単一ファイル（workflow companion）の同一セクション末尾への追記型であり、複数ブランチが並行して追記するとマージ時に同一アンカー行への挿入としてほぼ確実にコンフリクトする。ADR-0024 の Context はこの運用を想定していなかった。
2. **閾値条件の実績ゼロ**: 蓄積した実行記録8件（当初 `.pfdsl/workflow.md`、現在は `.pfdsl/bindings/pfd-retro.md` に分割済み）を確認すると、全件が「`/pfd-cycle` のサイクル完了」または「メタ設計対話のセッション締め」を起因としており、「新規設計決定記録2本以上」「同一 PFD への修正コミット3回以上」が単独トリガーとして発火した実績が1件もなかった。閾値条件は導入以来使われていない。

一方、pfd-ops プロトコル4は「作業完了 = 出力 artifact の status 更新（done への遷移）。コミットと同時に行う」と既に規定している。これは対象プロセス1件につき1回きりの機械判定イベントであり、`/pfd-cycle` の「毎サイクル終了後に pfd-retro が自動実行される」は実質このイベントに紐づいている。基準点を別途記録しなくても、done イベント自体が基準点として機能する。

## Decision

1. **実行記録を廃止する**: workflow companion への実行記録追記を止める。差分計測の基準点という記録目的そのものが (2) により消滅するため、記録先も不要になる。
2. **起動条件を done イベント駆動に一本化する**:
   - `/pfd-cycle` 経由: 対象プロセスの出力 artifact に done が付与された時点（プロトコル4）で pfd-retro を自動実行する。
   - 直接 pfd-ops を呼ぶ対話セッションで `/pfd-cycle` を起動していない場合: done イベントという基準点が存在しないため、この起動条件は適用対象外とする（サイクル終了トリガーを書いても意味を持たない）。
   - `/pfd-cycle` を起動した対話セッションでは、結論がまとまったタイミングで AI がユーザーに「そろそろサイクルを締めるか」を確認する。ユーザーの気付きを待つのではなく AI 側から能動的に問う。
3. 旧4条件（新規設計決定記録2本以上 / 同一 PFD への修正コミット3回以上 / 設計対話の長期化 / セッションの締め際）は削除する。

## Rationale

1. **効果が実績で確認できない記録機構は維持コストに見合わない**: 並列コンフリクトという構造的コストを払ってまで残す閾値条件が、一度も単独発火していない。
2. **既存の機械判定を転用する方が単純**: done イベントはプロトコル4に既に定義済みで、新しい記録機構を要らない。ADR-0024 の「基準点なしの閾値条件は死文」という問題意識自体は正しいが、解として記録を新設するのではなく、既存の done イベントを基準点として使えば記録は不要になる。
3. **対話セッションに無理にサイクル概念を持ち込まない**: `/pfd-cycle` を起動していない対話に「前回 retro からの差分」を計測させても、比較対象となる done イベントが存在せず判定不能。適用対象外と割り切る方が単純。

## Consequences

- pfd-retro `SKILL.md` 出力節の「実行記録」手順・D 層「休眠能力」の記述を削除・更新した。
- pfd-ops `SKILL.md` プロトコル「定期監査」を done イベント駆動の記述に書き換えた。
- `.pfdsl/bindings/pfd-retro.md` と scaffold `references/scaffold/bindings/pfd-retro.md` の「retro 実行記録」節を削除した（過去の記録は git 履歴に残る）。
- `scripts/check-companion-bindings.mjs` の必須見出しチェックから「retro 実行記録」を外した。
- ADR-0024 は Superseded by ADR-0026 とする（本文は編集せず Status 行のみ変更、改訂規約通り）。
