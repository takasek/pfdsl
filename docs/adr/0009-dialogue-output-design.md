# ADR-0009: 対話の出力設計 — 型付きシグナルと外化の分担

- Status: Accepted（2026-06-12 同日改訂: ADR 起草を独立プロセスに分離）
- Date: 2026-06-12

## Context

`docs/artifact_ecosystem.pfdsl`（生態系 PFD)で「ユーザーとの設計対話」をどうモデルするか、
4段階の変遷があった。

1. `dialogue` を不定形の外部入力 artifact とする — プロセスは立てず、対話はフローに現れない
2. `discuss` プロセスを立て、対話記録を出力とする — 出力が単一の曖昧な「記録」になり、
   4プロセスに消費されて発散した
3. 出力を型付けし、知見の外化（ADR・効果局面ログ）を対話プロセスに統合する —
   知見記録プロセスの入力集合が対話と一致し、分割が情報を増やさなかったため
   （ADR-0004 基準2）
4. ADR の一括起草を subagent に委譲した際、起草は判断（`decisions`）だけでは行えず、
   参照する図・成果物の実物との突合を要することが判明した（突合を欠いた起草は
   存在しないノード ID を捏造した）。ADR 起草は固有の入力と失敗様式を持つ
   独立プロセスとして再分離した — 現行設計

## Decision

対話はプロセス（`discuss`）として表現し、出力は型付きシグナル3種と軽量記録1種とする。

- **お題**（`topics`）: 次の実践に持ち込む題材
- **提案**（`proposals`）: issue 化候補となる改善案
- **判断**（`decisions`）: 方法論・設計に関する選択
- **効果局面ログ**（`payoff_log`）: 数行の事実追記。対話の終端動作として対話内で完結する

ADR は対話の直接出力ではない。`decisions >> draft_adrs -> adrs` —
構成的文書の起草は委譲可能な独立作業であり、参照する成果物の実物と
突合する検証を含む。

不定形のフロー外入力は `user_intent`（ユーザーの関心・意図）のみとする。
レビュー知見（`findings`）は `>>?` で次の対話へ還流する（通常エッジでは
お題→実践→レビューを経て循環するため）。

## Rationale

1. **外化されない議論は成果物を生まない**: ADR-0001 の出力有形性規則を対話自身に
   適用した帰結。議論がフロー内に留まる限り、後続プロセスが依拠できる artifact にならない。

2. **出力の型付けが発散を防ぐ**: 不定形の「対話記録」は受け取り手が解釈を強いられる。
   型付けすることで「この出力はどの消費先か」が決定論的になる。

3. **外化の重さで分担が決まる**: 効果局面ログは数行の事実記録で、対話の流れの中で
   完結する終端動作。ADR は構成・検証を要する文書執筆で、委譲・差し戻しという
   固有の作業実体を実際に持った。draft_adrs の分離は境界をまたぐ新依存
   （参照成果物との突合）と所有権境界（起草=subagent、評価=本体）を露出させる —
   ADR-0004 の基準2・基準4の両方が分割を支持する。

## Consequences

- 生態系 PFD の `discuss` は4出力、`draft_adrs` が ADR の単一生成元になる。
- `draft_adrs` は「参照する図・成果物の実物と ID・経路を突合する」検証を必須手順とする。
- `pfd-ops` スキルの知見振り分けプロトコルと整合 — 型の判断は対話中に完了し、
  ADR 級の判断は起草プロセスへ引き渡す。
