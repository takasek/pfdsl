# ADR-0009: 対話の出力設計 — 議論は外化までが一体

- Status: Accepted
- Date: 2026-06-12

## Context

`docs/artifact_ecosystem.pfdsl`（生態系 PFD）で「ユーザーとの設計対話」をどうモデルするか、
3段階の変遷があった。

1. `dialogue` を不定形の外部入力 artifact とする — プロセスは立てず、対話はフローに現れない
2. `discuss` プロセスを立て、対話記録を出力とする — 出力が単一の曖昧な「記録」になり、
   4プロセスに消費されて発散した
3. 出力を型付けし、知見の外化を対話プロセスの中に統合する — 現行設計

知見記録（ADR・効果局面ログへの書き込み）は当初独立プロセスとして分離していた。
しかし入力集合が対話と完全に一致し、境界をまたぐ新依存を生まなかった。
ADR-0004 基準2「情報を増やさない分割はしない」に照らして統合した。

## Decision

対話はプロセス（`discuss`）として表現する。出力は以下の型付きシグナルと永続記録で構成する。

- **お題**（`topics`）: 次の実践に持ち込む題材
- **提案**（`proposals`）: issue 化候補となる改善案
- **判断**（`decisions`）: 方法論・設計に関する選択記録
- **ADR**（`adr`）: 決定の恒久記録（知識記録クラスタ）
- **効果局面ログ**（`effect_log`）: 運用実績の定常記録（知識記録クラスタ）

記録（ADR・効果局面ログ）への外化は対話という一体の活動の終端動作であり、
独立プロセスではない。

不定形のフロー外入力は `user_intent`（ユーザーの関心・意図）のみとする。
レビュー知見（`findings`）は `>>?` で次の対話へ還流する（通常エッジでは
お題→実践→レビューを経て循環するため）。

## Rationale

1. **外化されない議論は成果物を生まない**: ADR-0001 の出力有形性規則を対話自身に
   適用した帰結。議論がフロー内に留まる限り、後続プロセスが依拠できる artifact にならない。

2. **出力の型付けが発散を防ぐ**: 不定形の「対話記録」は受け取り手が解釈を強いられる。
   3種のシグナルに型付けすることで「この出力はどの消費先か」が決定論的になる。

3. **記録は対話の終端動作**: 実践でも ADR・ログ追記は対話の流れの中で行われた。
   独立プロセスに分けると、対話終了後に「記録プロセス」を別途起動する意味になり、
   実態と乖離する。

## Consequences

- 生態系 PFD の `discuss` は5出力を持つ構造になる。
- `pfd-ops` スキルの知見振り分けプロトコルと整合する —
  「これは topics か proposals か decisions か」の判断は対話中に完了し、
  セッション外に持ち越さない。
- 新しい対話成果が生じたときは「どの型に当てはまるか」を問うことで消費先が決まる。
